# Theater of War

A browser-based turn-based strategy game played on a 3D hex grid. Built with Three.js, TypeScript, Vite, and Tailwind CSS — no backend, no install, runs entirely in the browser.

## Quickstart

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # production bundle
npm run preview  # serve the build locally
```

## How to Play

### Objective

Capture all enemy cities. You lose if all your cities fall.

### Turn Structure

Each turn resolves in order:

1. **Your turn** — move and attack with your units, queue production, upgrade buildings
2. **End Turn** — press the button (or `Enter`) to advance
3. **Enemy phase** — AI income, production, movement, and attacks resolve automatically
4. **Next turn begins** — your income and production tick, new units may spawn

### Controls

| Action | Input |
|--------|-------|
| Pan map | Left-click drag |
| Zoom | Scroll wheel |
| Select unit | Click unit |
| Move unit | Select → click highlighted hex |
| Attack | Select → click enemy (confirm on second click) |
| Deselect / close panel | `Esc` |
| End Turn | `Enter` or END TURN button |
| Cycle idle units | `Tab` or click the idle counter below END TURN |
| Navigate via minimap | Click anywhere on the minimap |

### Economy

- Each city you own generates **+30 $/turn**
- Markets add **+25 $/turn per level**
- Credits are capped at **500 + (Warehouse level × 250)**
- Production points are generated by factories: **+50 ⚙/level/turn**
- Units are built in a FIFO queue per city; only the front item advances each turn

### Fog of War

- Your **units** reveal a radius of 3 hexes
- Your **cities** reveal a radius of 2 hexes
- Enemy units outside your vision are hidden

## Units

| Unit | Requires | Cost | Prod | HP | Move | Notes |
|------|----------|-----:|-----:|---:|-----:|-------|
| Standard Infantry | Barracks lv1 | 50$ | 100⚙ | 100 | 2 | General purpose |
| Elite Infantry | Barracks lv2 | 100$ | 200⚙ | 120 | 2 | `anti_armor` |
| Armored Personnel Carrier | Factory lv1 | 120$ | 250⚙ | 120 | 4 | Fast, `transport` |
| Artillery | Factory lv1 | 150$ | 300⚙ | 80 | 1 | `indirect_fire`, min range 2, no counter |
| Medium Tank | Factory lv2 | 200$ | 400⚙ | 200 | 3 | `armored` |
| Heavy Tank | Factory lv3 | 400$ | 800⚙ | 350 | 2 | `armored`, `heavy_armor` |
| Fighter Jet | Airport lv1 | 300$ | 600⚙ | 100 | 6 | Air superiority |
| Strategic Bomber | Airport lv2 | 500$ | 1000⚙ | 120 | 5 | `aoe_blast_radius_1` |
| Destroyer | Harbor lv1 | 400$ | 800⚙ | 200 | 4 | Naval, range 2 |

| Tactical Nuclear Missile | Warehouse lv3 | 1000$ | 2500⚙ | 1 | 15 | `aoe_blast_radius_1`, one-shot |

Combat damage is type-matched: use the defender's movement type (`land`/`air`/`sea`) to pick the right damage column. Indirect-fire units attack without receiving a counter-attack.

Two friendly land units on the same hex **merge** into a stack (combined HP, shown as `×N`).

## Buildings

Click any player-owned city to open the city panel. Each building can be upgraded up to its maximum level.

| Building | Levels | Effect | Upgrade Costs |
|----------|--------|--------|---------------|
| Factory | 0–3 | +50 ⚙/level/turn production | 100 / 200 / 300 $ |
| Barracks | 0–3 | Unlocks infantry & APC | 80 / 160 / 240 $ |
| Warehouse | 0–3 | Credit cap +250$/level | 120 / 240 / 360 $ |
| Turret | 0–3 | Auto-fires at adjacent enemies each turn | 90 / 180 / 270 $ |
| Market | 0–3 | +25 $/level/turn income | 150 / 300 / 450 $ |
| Airport | 0–1 | Unlocks air units | 200 $ |
| Harbor | 0–1 | Unlocks naval units (coastal cities only) | 200 $ |

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Language | TypeScript (strict, ES2020) |
| 3D rendering | Three.js — orthographic camera, GLB models, CSS2D labels |
| Build | Vite + `tsc` |
| Styling | Tailwind CSS v4 |
| Game logic | Pure TS, zero DOM/Three.js dependencies in `src/engine/` |

### Architecture

```
src/
├── engine/          # Pure game logic — no DOM, no Three.js
│   ├── types.ts         # All shared interfaces and enums
│   ├── GameState.ts     # Single source of truth for runtime state
│   ├── TurnResolver.ts  # Full turn execution (income → production → AI → victory)
│   ├── EnemyAI.ts       # Stateless AI: production priority + move/attack logic
│   ├── CombatResolver.ts# Simultaneous damage resolution
│   ├── Pathfinder.ts    # BFS reachability + A* pathing + attack targeting
│   ├── HexUtils.ts      # Axial coordinate math
│   └── MapGenerator.ts  # Static 37-cell map with 7 cities
│
├── renderer/        # Three.js — reads state, never writes it
│   ├── SceneManager.ts  # Camera, lights, render loop
│   ├── HexRenderer.ts   # Hex cell meshes, fog, range overlays
│   ├── UnitRenderer.ts  # Unit models, HP labels, animations
│   ├── InputManager.ts  # Raycasting mouse → hex callbacks
│   ├── ModelLoader.ts   # GLB cache + clone + auto-normalize
│   └── constants.ts     # hexToWorld(), HEX_SIZE, terrain/owner colors
│
├── ui/              # DOM panels
│   ├── CityPanel.ts     # Slide-in right panel: buildings + production queue
│   └── dom.ts           # el() and btn() helpers
│
└── main.ts          # Wiring: orchestrates all three layers
```

Map seeds are preserved in the URL (`?seed=N`) so any game can be replayed or shared.
