# 📜 SYSTEM ARCHITECTURE & GAME DESIGN DOCUMENT

**Project:** "Theater of War" (Working Title)

**Target Platform:** Browser-based Singleplayer Game (Client-only)

**Tech Stack:** TypeScript (Strict Mode), Vite, Three.js (3D Rendering), TailwindCSS (UI Layer)

**Design Principle:** Data-driven, component-based, high scalability for future content additions.

---

## 1. GAME OVERVIEW & CORE LOOP

The game is a zoomed-out, fast-paced military strategy game with a modern "tabletop/boardgame" aesthetic (low-poly 3D models acting as oversized pieces on a stylized hex grid, inspired by the scale in `image_4a27a7.jpg`).

1. **Build & Produce:** Click on owned cities to upgrade infrastructure (Barracks, Factories, Warehouses) and produce units using accumulated **Credits** ($) and **Production** points (⚙️).
2. **Deploy & Move:** Move units across the hexagon map (different unit types have different terrain pathfinding).
3. **Execute & Fight:** Press "End Turn". The global game loop resolves income, production progress, AI movements, and calculated combat encounters.
4. **Win Condition:** Capture all enemy cities or destroy the enemy's main headquarters.

---

## 2. ECONOMIC & LOGISTICS SYSTEM

To keep the game fast and accessible, there are only two global resources, regulated by city infrastructure limits:

- **Credits ($):** Generated passively by cities every turn. Used to purchase units and building upgrades.
- **Production (⚙️):** Generated by Factories. Determines how many turns a unit or building upgrade takes to complete.
- **Global Unit Capacity:** Regulated by the total level of Barracks and special military structures. You cannot build more units than your capacity allows.

### City Infrastructure (Building Levels 0 to 3)

- **Factory (⚙️):** Increases production output per turn. Higher levels unlock heavier, advanced machinery blueprints.
- **Barracks (🪖):** Increases global Land Unit capacity. Unlocks advanced infantry/vehicle blueprints.
- **Warehouse (📦):** Increases max resource storage capacity, preventing resource capping. Required to hoard enough capital for high-end weapons (e.g., Nukes).
- **Airport / Harbor (✈️/⚓):** Specialized static structures required to spawn Air or Sea units on adjacent matching tiles.
- **Defense Turret (🛡️):** Automated city defense. Automatically damages adjacent enemy units during the upkeep phase.

## 3. DATA-DRIVEN UNIT MATRIX (SCALABILITY CORE)

The AI Agents **must not** hardcode specific units. All units are defined as data objects within a central registry (`units.json` configuration style). This allows instant scaling for future units like Submarines, Aircraft Carriers, or ICBMs.

### Unit Data Structure Example

```json
[
  {
    "id": "infantry_basic",
    "name": "Standard Infantry",
    "requiredBuilding": "barracks",
    "requiredBuildingLevel": 1,
    "cost": { "credits": 50, "productionNeeded": 100 },
    "movement": { "type": "land", "range": 2 },
    "combat": {
      "damageVsLand": 15,
      "damageVsAir": 2,
      "damageVsSea": 0,
      "range": 1
    },
    "specialTraits": []
  },
  {
    "id": "ballistic_nuke",
    "name": "Tactical Nuclear Missile",
    "requiredBuilding": "warehouse",
    "requiredBuildingLevel": 3,
    "cost": { "credits": 1000, "productionNeeded": 2500 },
    "movement": { "type": "ballistic", "range": 15 },
    "combat": {
      "damageVsLand": 200,
      "damageVsAir": 200,
      "damageVsSea": 200,
      "range": 1
    },
    "specialTraits": ["aoe_blast_radius_1", "consume_on_attack"]
  }
]
```

### Movement Rules (Pathfinding via A\* Algorithm)

- `land`: Can pass through Plains and Forests. Blocked by Water and Mountains.
- `water`: Can only pass through Water tiles. Blocked by all Land and Mountains.
- `air`: Can pass through all tiles. Must land/end turn on a friendly City, Airport, or Aircraft Carrier (if implemented).
- `ballistic`: Ignores all pathfinding. Directly travels to target coordinate within range and triggers impact effect.

## 4. TECHNICAL STATE ARCHITECTURE

The TypeScript architecture must clearly separate **Game Logic (State)** from **Visual Rendering (Three.js)**.

### 4.1 Hex Cell State

```typescript
interface HexCell {
  id: string; // Axial coordinate notation "q_r"
  q: number;
  r: number;
  terrain: "plains" | "forest" | "mountain" | "water";
  cityId: string | null; // References City object if present
  unitId: string | null; // References MilitaryUnit present on cell
}
```

### 4.2 City State

```typescript
interface City {
  id: string;
  name: string;
  owner: "player" | "enemy" | "neutral";
  hexId: string;
  buildings: {
    factoryLevel: number;
    barracksLevel: number;
    warehouseLevel: number;
    airportLevel: number;
    harborLevel: number;
    turretLevel: number;
  };
  productionQueue: {
    blueprintId: string; // Unit or Building ID being built
    progressPoints: number;
  }[];
}
```
